We control the behavior of Semagrams using an asynchronous IO monad from cats-effect.

We control the behavior of Semagrams using an asynchronous IO monad from cats-effect.

This allows the "state" of Semagrams to simply be a continuation for receiving the next event. This makes things like modal interfaces and multi-keypress commands very easy; rather than explicitly representing the state machine as an enum, we can simply write something like

for {
 source <- selectSource()
 target <- selectTarget()
 makeEdge(source, target)
} yield ()

where selectSource and selectTarget might involve arbitrary interaction of the user with Semagrams.
